# 里程碑计划（MVP 组合 B）

> **所属文档**：方案设计 · 子模块  
> **版本**：v1.0  
> **范围**：7 个 MVP 模块（含八字完整版：命盘 + 五行 + 十神 + 大运 + 流年）

## 1. 里程碑总览

| 里程碑 | 时长 | 产物 | 可验收标准 |
|-------|------|------|-----------|
| **M0 启动** | 0.5 天 | 仓库初始化、工具链就绪 | `pnpm dev` 可启动 |
| **M1 基础设施** | 2 天 | 路由、布局、Store、Disclaimer、主题系统 | 空白 7 个页面可点击 |
| **M2 轻量模块** | 4 天 | 称骨、小六壬、灵签、解梦 | 4 个模块可完整使用 |
| **M3 八字完整版** | 6 天 | 命盘 + 大运 + 流年 | `BaziView` 全功能可用，含时间轴 |
| **M4 姓名学 + 择日** | 3 天 | 姓名、黄历 | 完整 7 模块 |
| **M5 数据层工程** | 5 天（并行）| 采集 + 模板生成 + 审校 | 所有 C 类文案 100% 审校完成 |
| **M6 UI/UX 打磨** | 3 天 | 动画、分享卡、暗色模式 | 视觉定稿 |
| **M7 性能与测试** | 2 天 | Lighthouse、兼容性、无障碍 | 所有指标达标 |
| **M8 部署上线** | 1 天 | 域名、CDN、监控 | 线上可访问 |

**合计：约 26-30 个工作日**（单人），**并行 2 人约 14-18 天**。

## 2. 分阶段详细任务

### M0 启动（0.5 天）

- [ ] `pnpm create vite tt-qimen --template vue-ts`
- [ ] 安装依赖：`vue-router`、`pinia`、`@vueuse/core`、`tyme4ts`、`chinese-character-strokes`、`fuse.js`、`html2canvas`、`sass`
- [ ] 配置 `vite.config.ts`（别名、代理、压缩插件）
- [ ] 配置 ESLint + Prettier + Husky pre-commit
- [ ] 创建目录骨架（`src/pages/*`、`src/core/*`、`src/data/*`、`src/components/*`）

### M1 基础设施（2 天）

- [ ] `App.vue` 整体布局（顶栏 + 内容 + 底栏）
- [ ] `AppHeader.vue`、`DisclaimerBanner.vue`、`AppFooter.vue`
- [ ] 路由懒加载配置（7 个页面）
- [ ] Pinia：`useUserStore`（生辰信息本地缓存）、`useThemeStore`
- [ ] Sass 变量文件（国风色板 + 现代简约色板切换）
- [ ] 全局错误边界 + 404 页

### M2 轻量模块（4 天）

按"最小可用"顺序实现：

- [ ] **Day 1: 小六壬**（最简单，0.5 天算法 + 0.5 天 UI）
- [ ] **Day 2: 称骨算命**（0.5 天算法 + 0.5 天动画）
- [ ] **Day 3: 观音灵签**（0.5 天采集清洗 + 0.5 天摇签动画）

### M3 八字完整版（6 天）

## 5. 风险登记表

| 风险 | 级别 | 缓解措施 |
|------|-----|---------|
| **流年模板 300 段审校超时** | 高 | 先做 60 段高频组合，其余用"60 甲子通用模板"兜底 |
| **`tyme4ts` 生僻算法与传统口诀不一致** | 中 | 对照 6tail 官网的命盘 demo 验证（已有先例）|
| **iOS Safari 动画性能不足** | 中 | 使用 CSS transform + will-change，避免 JS 驱动 |
| **`html2canvas` 在微信中字体丢失** | 中 | 关键字体内联 + fallback 系统字体 |
| **LLM 审校进度滞后** | 高 | 先上线"降级文案"（模板占位），T+7 再替换完整文案 |
| **观音灵签 API 不稳定** | 低 | 一次性爬取落库，不依赖运行时 API |
