## 1. 基础约定

### 1.1 响应式断点

| 断点 | 宽度范围 | 典型设备 | 布局策略 |
|------|---------|---------|---------|
| **Desktop** | ≥ 1024px | 电脑、平板横屏 | 多列平铺，内容居中 1024px |
| **Tablet** | 768px – 1023px | 平板竖屏、折叠屏展开 | 2 列或关键区双列 |
| **Mobile** | ≤ 767px | 手机 | 单列滚动，高优先级内容优先展示 |

### 1.2 坐标系与画幅

本文档的 ASCII 线框图宽度统一为 **62 字符**（约对应 Desktop 的"内容区"宽度缩影），移动端用 **38 字符** 画幅，两者在同一模块并列显示以对照折叠规则。

### 1.3 符号图例

| 符号 | 含义 |
|------|-----|
| `[文本]` | 可点击按钮、链接、标签 |
| `[...提示文字...]` | 输入框，`...` 为 placeholder |
| `☐` / `☑` | 未选 / 已选（复选、单选） |
| `★★★★☆` | 吉日星级、签文等级 |
| `🟢 🟡 🔴` | 吉 / 中 / 凶 状态色条 |
| `← →` | 水平滚动 / 导航 |
| `▼ ▲` | 折叠 / 展开 |
| `···` | 省略内容或懒加载占位 |
| `—+—` | 分隔线 |

## 3. 核心复用组件

### 3.1 生辰输入器（`components/BirthInput.vue`）

**使用模块**：八字、称骨、小六壬（部分）

**Desktop 折叠态**：

```text
┌──────────────────────────────────────────────────────────────┐
│  ✍️ 请输入您的生辰                                            │
│ ──────────────────────────────────────────────────────────── │
│  出生日期：[2000-01-01 ▾]   出生时间：[12:30 ▾]              │
│  性别：   (●) 男   ( ) 女                                    │
│  ☐ 使用真太阳时（按出生地经度调整，高级选项）                 │
│                                                              │
│                              [  开始排盘  ]                   │
└──────────────────────────────────────────────────────────────┘
```

**Mobile**（字段纵排）：

```text
┌──────────────────────────────────────┐
│  ✍️ 请输入您的生辰                    │
│ ──────────────────────────────────── │
│  出生日期                            │
│  [ 2000-01-01  ▾ ]                   │
│  出生时间                            │
│  [ 12:30  ▾ ]                        │
│  性别                                │
│  (●) 男   ( ) 女                     │
│  ☐ 使用真太阳时                      │
│                                      │
│  [       开始排盘       ]            │
└──────────────────────────────────────┘
```

**交互细节**：
- 日期/时间选择使用原生 `<input type="date">` / `type="time">`，避免引入大型 UI 库
- 真太阳时默认关闭，展开后显示文字提示"按出生地经度 ±10 分钟"
- 表单校验：未填 → 按钮禁用；过于未来日期 → 红色提示
- 数据持久化：成功排盘后写入 Pinia 的 `userStore.birth` + localStorage
- 切换页面后回到此组件会**自动回填**上次的值

### 3.2 结果卡片（`components/ResultCard.vue`）

通用结果容器，所有模块共用。结构：

```text
┌──────────────────────────────────────────────────────────────┐
│  🎯 结果标题（如"您的命盘""小六壬结果""第 38 签"）             │
│ ──────────────────────────────────────────────────────────── │
│                                                              │
│  ……结果内容（由子组件填充）……                                │
│                                                              │
│ ──────────────────────────────────────────────────────────── │
│  [再测一次]        [分享结果]         [收藏（本地）]          │
└──────────────────────────────────────────────────────────────┘
```

### 3.3 分享弹窗（`components/ShareModal.vue`）

**弹出状态（带遮罩）**：

```text
┌══════════════════════════════════════════════════┐
║                 分享结果                     [×] ║
║ ──────────────────────────────────────────────── ║
║                                                  ║
║    ┌────────────────────────────┐                ║
║    │                            │                ║
║    │    （分享预览卡片）         │                ║
║    │    国风底纹 + 结果摘要      │                ║
║    │    启门水印                │                ║
║    │                            │                ║
║    └────────────────────────────┘                ║
║                                                  ║
║   [复制链接]  [保存为图片]  [复制口令]            ║
║                                                  ║
║   ⚠️ 此卡片由 HTML2Canvas 在本地生成，不上传      ║
╚══════════════════════════════════════════════════╝
```

**交互细节**：
- 点击遮罩或 `[×]` 关闭
- "复制链接"→ 将带 URL 参数的结果页链接写入剪贴板（如 `?birth=2000-01-01T12:30`）
- "保存为图片"→ 调用 html2canvas（或原生 Canvas）生成 PNG 并触发浏览器下载
- "复制口令"→ 生成类似淘口令的短文本（如"『甲子 乙丑 丙寅 丁卯』— 启门 tt-qimen.com"）
- ESC 关闭弹窗

#### 4.1.3 首页交互细节

| 元素 | 行为 |
|------|------|
| 小六壬卡片（Desktop）| 使用"系统今天"自动排出当日宫位，无需用户输入；点击 [查看详情 →] 跳转 `/liuren` 并携带今日参数 |
| 小六壬卡片（Mobile）| 同上，但不展示 6 宫图，只显示命中宫位 + 一句话解读 |
| 黄历宜忌卡片 | 调用 `tyme4ts` 获取"今日"的宜/忌/冲/煞，点击跳转 `/zeri`，进入月历视图时保持"今日"高亮 |
| 八字招牌卡片 | 点击后若 localStorage 有生辰信息则直接排盘，否则进入生辰输入页 |
| 6 个小卡片 | 点击跳转对应路由；每张卡片右下角显示预估耗时（1/2/3 分钟）便于用户决策 |
| Mobile 顶部抽屉 | 打开后从左滑入，覆盖 80% 屏宽，右侧保留遮罩；点击遮罩关闭 |
| 深色模式切换 | 立即生效，无动画闪屏（CSS 变量全局替换）|

#### 4.2.4 八字页交互细节

| 元素 | 行为 |
|------|------|
| [修改生辰] | 展开 BirthInput，保留当前结果在下方；修改后点"重新排盘"覆盖旧结果 |
| 五行雷达图 | 数值从 0 动画增长到目标值（Vue transition），避免"一瞬间出现"的生硬感 |
| 十神结构区 | 年/月/时 三行默认展开；"详细十神解读 ▼"点击后展开完整 10 种十神含义（懒加载 tenGods.ts）|
| 大运时间轴 | 桌面 5 列 2 行蛇形网格（→/↓/←），移动 2 列 5 行蛇形；卡片间虚线连接表达顺序；当前大运高亮（比对当前年龄），不再横向滚动 |
| 大运格子 | 点击任一格：下方展开"一句话提示"（从 fortuneHints.ts 查表）；不做展开式多段解读 |
| 大运吉凶标签 | 颜色编码：吉=浅金、中=浅灰、凶=浅朱；避免大红大绿（规避用户过度解读） |
| 流年区块 | 默认显示当前年份（如 2026）；"查看 2027 流年 ▾"下拉展开后续 10 年；不显示月/日细节 |
| [分享命盘] | 调起 ShareModal，预览卡片包含四柱 + 五行 + 日主 + 当前大运 + 主题纹样 |
| [再排一次] | 滚动到顶部并清空结果；保留生辰信息，方便换"真太阳时"对比 |
| 错误态 | 未来日期、非法日期时，按钮 disabled + 红色提示"请输入合法的历史日期（1900–至今）" |

## 5. 关键交互细节

### 5.1 生辰输入：首次 vs 回访

```text
首次访问八字页：
   进入页面 → 显示空白生辰输入器 → 用户输入 → [开始排盘] → 结果

回访八字页（localStorage 有数据）：
   进入页面 → 自动展示上次结果 + 小卡片「您上次的生辰: 2000-01-01 12:30 男」
          → 用户想改时点「修改生辰」→ 表单展开 → 同上
```

### 5.2 空态 / 加载态 / 错误态

每个主要模块都要提供以下三态：

```text
加载态（懒加载 JSON 或复杂排盘时）：
   ┌──────────────────────────────┐
   │     ⏳ 正在排盘…              │
   │   tyme4ts 初始化中…           │
   └──────────────────────────────┘

空态（如择日无结果）：
   ┌──────────────────────────────┐
   │     😶 未找到符合条件的吉日   │
   │  尝试放宽事由或扩大时间范围   │
   │   [调整条件]                 │
   └──────────────────────────────┘

错误态（如日期越界）：
   ┌──────────────────────────────┐
   │ ⚠️ 仅支持 1900-01-01 至今     │
   │      请重新输入出生日期       │
   │          [返回修改]          │
   └──────────────────────────────┘
```

### 5.3 分享卡片生成流程

```text
用户点击 [分享] → 弹出 ShareModal
   ↓
ShareModal 内部调用 html2canvas 渲染预览区
   ↓
预览完成（~300ms） → 显示 3 个操作按钮
   ↓
[复制链接] → 写剪贴板 → 「已复制，快去粘贴分享吧」轻提示
[保存图片] → 触发 a.download → 生成 `启门-命盘-2026-04-18.png`
[复制口令] → 写剪贴板 → 显示示例效果
```

### 5.4 数据持久化规则

| 数据 | 存储位置 | 保留时长 | 清理时机 |
|-----|---------|---------|---------|
| 生辰信息 | localStorage | 永久 | 用户手动"清空"或清浏览器数据 |
| 姓名 | localStorage | 永久 | 同上 |
| 历史签文 | localStorage（最多 10 条）| 永久 | FIFO 自动覆盖 |
| 深色模式 | localStorage | 永久 | — |
| 当前排盘结果 | Pinia（内存） | 当前会话 | 关闭 Tab 即失 |

### 5.5 深色模式

统一使用 `[data-theme="dark"]` 切换 CSS 变量；所有组件不单独实现深色样式。

```text
浅色 ┼ 深色
──────┼──────
#FDF6E3 │ #1C1917   (背景)
#1C1917 │ #FDF6E3   (文字)
#B91C1C │ #F87171   (主色 红)
```
