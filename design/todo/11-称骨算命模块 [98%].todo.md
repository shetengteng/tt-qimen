## 附录 F · FortuneError 集成口径（2026-04-26）

`core/chenggu.ts::calculateChenggu()` 内三处异常入口统一抛 `FortuneError`：

| 触发条件 | code | 含义 | details 字段 |
|---------|------|------|------------|
| 年/月/日/时 4 张权重表任一查不到 | `'invariant'` | 理论不会触发的查表缺失（114 条已穷举所有合法取值） | `{ yearGanzhi, monthNum, dayNum, hourIndex, hasY, hasM, hasD, hasH }` |
| 总骨重 < 2.1 或 > 7.1（容差 ±0.01） | `'out-of-range'` | 越界，UI 显示排盘失败 | `{ total, min, max, breakdown: { y, m, d, h } }` |
| 51 首歌诀里没有匹配的总骨重 | `'invariant'` | 51 段已覆盖 [2.1, 7.1] 所有 0.1 间隔，理论不该发生 | `{ total, breakdown: { y, m, d, h } }` |

UI 层 `ChengguPage.vue::onPaipan()` 的 try/catch 不感知具体 code，统一降级到 `chenggu.computeError.*` 卡片；`FortuneError` 的 `module / code / details` 由开发者在 console 看到，未来可对接全局错误上报。

## 附录 G · 三语 i18n 落地策略（2026-04-26）

整体复用 liuren 模块的 `palacesLocale` 模式 —— **核心是把 locale 放进 core 计算函数，输出已本地化的 result，UI 层不再感知**。

### 文本字段一览

| 字段 | zh-CN | zh-TW | en |
|------|-------|-------|-----|
| `poem.weight`（"三两一钱"） | 简体直出 | `tify()` 转繁体（"三兩一錢"） | 简体直出（不译） |
| `poem.poem[]`（4 句歌诀） | 简体直出 | `tify()` 转繁体 | 简体直出（古文文化属性强，禁意译） |
| `poem.description`（白话） | 简体直出 | `tify()` 转繁体 | 简体直出 |
| `displayWeight` | 简体 | 繁体 | 简体 |
| `breakdown.{year,month,day,hour}.label` | "己巳年/五月生/十九日/子时生" | "己巳年/五月生/十九日/子時生" | 同 zh-CN |

### 实现要点

1. **`core/chenggu.ts::calculateChenggu(birth, locale = 'zh-CN')`**：新增 locale 参数，仅影响 result 文本字段；数值 / level / weightNumber 等数据本体不变。
2. **`data/poemsLocale.ts::getLocalizedPoem(weightNumber, locale)`**：按 weightNumber 缓存繁体版本，避免每次切 aspect 都重转 51 首歌诀。
3. **`ChengguPage.vue`**：传入 `localeStore.id`；`watch(localeStore.id)` 在已有 result 时自动重算（不重新动画，纯数据级 i18n 切换）。
4. **`PoemDisplay.vue` / `WeightTable.vue` / `InterpretBlock.vue`**：保持纯展示组件不感知 locale，直接消费 `result.poem.*` / `result.breakdown.*`。

### 为什么 en 不译歌诀？

设计文档 §8 明确：袁天罡称骨歌是中国传统命理术语高度浓缩的古文，意译会丢失"骨重 - 命格"的对应关系（"二两一钱 = 短命非业谓大空" 这种对仗逻辑）。en 模式下 description 也保留中文，是用户口径的明确决定（与 liuren 36 句解读不同 —— liuren 解读是现代散文，可英译；chenggu 歌诀是古诗，不可意译）。
