# 周公解梦 · 现代解读草稿（drafts）

> 本目录（`design/jiemeng/drafts/`）存放 `raw/*.md` 中每条词条对应的
> **现代解读字段**（`summary` / `modern` / `advice`），与 raw 严格分层：
>
> - `design/jiemeng/raw/` — 公版古籍原文（无版权、不改写）
> - `design/jiemeng/drafts/` — 现代化解读（编辑撰写、可迭代）
> - `src/modules/jiemeng/data/dreams.generated.ts` — 由 `scripts/jiemeng/build-dreams.mjs` 合并 raw + drafts 后生成

## 1. 与 raw 的关系

drafts 中的每个 `.md` 与 raw 中**同名文件一一对应**（`01-animal.md` ↔ `01-animal.md`）。
build 脚本通过 `### {title}` 标题匹配。

- raw 中**有但 drafts 中无对应**的词条 → 字段留空（`summary=""` / `modern=[]` / `advice=""`），build 不报错，便于分批推进。
- drafts 中**有但 raw 中无对应**的词条 → build 报警告，避免 typo 导致草稿丢失。

## 2. 词条格式

```markdown
### 梦见蛇咬自己

**summary**: 古籍云：梦被蛇咬，主得财货。现代心理学认为多与情绪压力、人际冲突有关。

**modern**:
- 从传统象征看，梦见蛇咬自己是"得财"之兆，古人认为这是财运或贵人即将到来的信号。
- 从现代心理学角度，蛇在梦中常代表难以处理的情绪或关系中的紧张张力。被蛇咬可能提示近期的压力积累。

**advice**: 最近若有重要决定，可放缓一步、多咨询身边信任的人；情绪紧绷时，不妨以写日记或冥想来疏导。
```

### 字段约束

| 字段 | 形态 | 字数建议 | 说明 |
|------|------|---------|------|
| `summary` | 单段 | 30-80 字 | "古籍云：…。现代解读为…。" 一句话浓缩，列表预览用 |
| `modern` | 2-4 项 bullet | 每段 60-150 字 | 第 1 段从传统象征切入；第 2 段从现代心理学切入；可加第 3 段补充情境 |
| `advice` | 单段 | 30-80 字 | 给"今天醒来的梦者"一句温和、可执行的建议，不下吉凶判断 |

## 3. 撰写原则

1. **古籍象征 + 现代心理学双轨**：尊重古文吉凶倾向，但用心理学语言中性化（如"得财"→"贵人或机遇预兆"，"主死"→"重大转变的象征"）。
2. **避免吉凶决断**：不能写"必有大吉/必有灾祸"，改为"古人视此为…的征兆"。
3. **避免医疗/财务/法律建议**：不能写"应去医院"或"建议买入"，改为"可关注身体信号"或"留意财务节奏"。
4. **敏感词条采用文化象征视角**：death / illness / spirit 类词条，advice 重在"理解情绪"而非"消灾"。
5. **每条独立可读**：不假设用户已读过其他词条。

## 4. Codex 审校须知

提交给 Codex 校验时建议附带的 prompt 模板：

```text
请按以下规则审校 design/jiemeng/drafts/01-animal.md，目标：
1. summary 是否在 30-80 字、是否真实概括了同名 raw 词条的古文要旨
2. modern 是否做到「传统象征」+「现代心理学」双视角，且未下吉凶决断
3. advice 是否温和、可执行、不涉及医疗/财务/法律
4. 整体语气是否与现有 38 条手写样本（src/modules/jiemeng/data/dreams.ts）一致

对每条问题给出：
- 词条 id（如 snake-bite）
- 问题分类（违反原则 / 字数超限 / 与古文不符 / 语气不一致）
- 建议修订
```
