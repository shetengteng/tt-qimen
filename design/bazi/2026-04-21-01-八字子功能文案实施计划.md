## 0. 核心约束（不可妥协）

本计划遵循 `user_rule · 生产级任务执行协议`：

1. **最小化变更**：每轮只动一个文案类别，不合并多类
2. **局部化**：新增 `src/modules/bazi/data/*.ts`，不改 core / components / page
3. **不重构**：不动既有 `PATTERN_COMMENT`（pattern.ts）、`PATTERN_HINT`（interpretTemplate.ts）——**新增不替代**
4. **版本标注**：每个 .ts 文件头部注明古籍出处 + 采录日期
5. **字数控制**：按调研文档规定（短 ≤20 字、长 80-120/200 字）
6. **审校责任**：白话改写后必经人眼二校，禁止直译

## 2. 实施方案

### 阶段 A · 纳音含义（30 段）

**目标文件：** `src/modules/bazi/data/nayin.ts`（新建）

**素材位置：** `design/bazi/extracted/01-nayin.md`（已有 30 条原文引用 + 象意要点 + short 占位）

**工作步骤：**

1. **定稿 short 文案（30 条）**
   - 每条 ≤20 字
   - 格式：`{意象关键词}，{引申含义}`，如"藏锋蓄锐，动则见光"
   - 已在 `01-nayin.md` 中有占位稿，需审校

2. **撰写 long 文案（30 条）**
   - 每条 80-120 字
   - 三段式结构：
     - 句 1：点明纳音意象（源自古籍）
     - 句 2：归纳性格/命运的 2-3 个关键词
     - 句 3：最宜 / 最忌的行运或五行配合

3. **创建 TypeScript 数据文件**

   ```typescript
   // src/modules/bazi/data/nayin.ts
   /**
    * 纳音 30 种含义
    *
    * 原文来源：《三命通会·卷一·论纳音取象》（维基文库版 2026-04-20 采录）
    * 白话改写：2026-04-21
    */
   export interface NayinMeaning {
     short: string   // ≤20 字，四柱表下方直接显示
     long: string    // 80-120 字，tooltip / 详情页
     element: '金' | '木' | '水' | '火' | '土'
   }

   export const NAYIN_MEANING: Record<string, NayinMeaning> = {
     海中金: { ... },
     炉中火: { ... },
     // ... 共 30 条
   }

   export function getNayinMeaning(nayin: string): NayinMeaning | null {
     return NAYIN_MEANING[nayin] ?? null
   }
   ```

4. **接入 UI**
   - 位置 1：`FourPillarsTable.vue` 第 149 行 `<div class="bazi-cell-nayin">{{ p.nayin }}</div>` 增加 tooltip
   - 位置 2：命盘解析第一段适当引用

**验收标准：**
- [ ] 30 条 short 文案全部 ≤20 字，意象明确
- [ ] 30 条 long 文案 80-120 字，可独立成段
- [ ] TypeScript 编译通过，lint 无警告
- [ ] FourPillarsTable 悬浮时能看到 long 解读

**预估工时：** 1 天（主要在审校 + 白话改写）

- 辅源：`design/bazi/raw/sanming-tonghui/volume-10-12-extracts.md`（赋文纲要）

**工作步骤：**

1. **分章定位原文**
   按调研文档 § 2.4 列表，从《子平真诠》正文抽取 11 个格局对应的段落：
   - 论正官 / 论七杀（偏官）/ 论正财 / 论偏财
   - 论印绶（正印）/ 论偏印 / 论食神 / 论伤官
   - 论建禄月劫（含建禄、月刃）
   - 论杂气 / 特殊格（从化格归入"杂气格"字段）

2. **每格抽取 200 字白话长解读**
   结构：
   - 句 1-2：格局定义 + 喜忌大方向（80 字）
   - 句 3-4：成格条件、破格条件（70 字）
   - 句 5：行运影响（50 字）

3. **创建数据文件**

   ```typescript
   // src/modules/bazi/data/patternLongInterpret.ts
   /**
    * 11 格局长解读
    *
    * 原文来源：《子平真诠评注》+《三命通会·卷八》(ctext.org/维基文库 2026-04-20 采录)
    * 白话改写：2026-04-21
    */
   import type { PatternName } from '../types'

   export const PATTERN_LONG_INTERPRET: Record<PatternName, string> = {
     正官格: '...',  // 200 字
     七杀格: '...',
     // ... 共 11 条
   }

   export function getPatternLongInterpret(pattern: PatternName): string {
     return PATTERN_LONG_INTERPRET[pattern] ?? '该格局以常规论，详见喜用神配合。'
   }
   ```

4. **接入 UI**
   - **新增**：命盘解析第二段下方"查看完整分析"展开区
   - **不替换**：现有 `PATTERN_HINT` 短点评保留，作为摘要

**验收标准：**
- [ ] 11 条长解读每条 200 字（±20），分三段结构清晰
- [ ] 每条都可溯源到《子平真诠》或《三命通会》对应章节
- [ ] 展开区 UI 交互顺畅（折叠态/展开态切换）

## 5. 风险与约束

### 5.1 版权合规

- 所有原文均来自作者去世 > 100 年的公版古籍（详见 `design/bazi/raw/README.md`）
- 白话改写属于"对公版内容的转述"，不构成侵权
- 每个 .ts 文件头部必须注明出处 + 采录日期

### 5.2 文风统一

- 纳音、格局、神煞三类文案必须保持一致的"客观陈述、不算命"口吻
- 避免"必、定、绝对、死"等绝对化措辞
- 使用"主、宜、防、可"等调和措辞（与现有 `fortuneHints.ts` 一致）

### 5.3 i18n 预留

- 所有中文字段统一用中文键（`海中金: {...}`），与繁体神煞 key 保持一致的思路
- 未来要出英文版时，另建 `nayin.en.ts`，按相同结构替换
