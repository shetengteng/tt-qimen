# 男女同生辰多页面差异验证报告

> 日期：2026-04-19
> 范围：八字（tyme4ts）/ 紫微斗数（iztro）
> 目的：回应"男女同辰为何看上去命一样"以及"切性别五行强弱不变是否正常"两个问题，并以实测数据印证。

---

## 1. 结论摘要

| 问题 | 结论 |
|---|---|
| 男女相同生辰，"命"是否相同？ | **是相同的**（四柱、五行、十神、宫位、星曜、四化，皆为客观气场，性别不参与） |
| 性别会改变什么？ | **是"运"** —— 大运/大限的"起运岁数"和"顺逆方向"由性别 + 年干阴阳决定 |
| 切性别后五行强弱不变是否正常？ | **正常** —— 它属于命，本就与性别无关 |
| 之前 UI 看着"完全一样"，是 bug 吗？ | 是 UI/交互层的两个 bug：① 首屏未排盘，渲染的是 mock 默认值；② 标题硬编码"乾造"且性别变化没有联动重排 |

---

## 2. 原理：命 vs 运

### 2.1 八字（tyme4ts 框架）

| 维度 | 是否取决于性别 | 出处 |
|---|---|---|
| 年/月/日/时 四柱（八个字） | 否 | `SolarTime.fromYmdHms(...).getLunarHour().getEightChar()` |
| 五行强弱（量化） | 否 | 仅基于四柱天干地支 |
| 十神 / 纳音 / 神煞 | 否 | 同上 |
| **大运起运岁数** | **是** | `ChildLimit.fromSolarTime(solar, gender)` |
| **大运行运方向（顺/逆）** | **是** | 阳男阴女顺行；阴男阳女逆行 |
| 流年（基于大运推） | 间接是 | 因起运点不同而不同 |

代码印证：

```141:147:src/modules/bazi/core/bazi.ts
const childLimit = ChildLimit.fromSolarTime(
  solar,
  birth.gender === 'male' ? Gender.MAN : Gender.WOMAN,
)
```

### 2.2 紫微斗数（iztro 框架）

| 维度 | 是否取决于性别 | 出处 |
|---|---|---|
| 十二宫位排布 | 否 | 由月支、时支推 |
| 14 主星 / 副星 / 煞星位置 | 否 | 由日干、年干推 |
| 四化飞星（化禄/权/科/忌） | 否 | 由年干推 |
| **大限起运与方向** | **是** | "阳男阴女顺行，阴男阳女逆行"分流 |
| 小限起点 | 间接是 | 因大限分流不同 |

## 3. 之前 UI "完全一样"的根因

### 根因 A：首屏未排盘，看到的是 mock 默认值

`BaziPage.vue`、`ZiweiPage.vue` 在用户输入并点"开始排盘"前，仅渲染 `chart.value` 为空时的 mock 数据。即使用户在 Setting 中改了性别，首屏依然是同一份固定预览。

**修复**：在 `onMounted` 中，若 `userStore.birth` 不是默认值，自动 `onPaipan()`，让首屏一打开就看到真实计算结果。

```158:162:src/modules/bazi/BaziPage.vue
onMounted(() => {
  if (!userStore.isDefault) {
    onPaipan()
  }
})
```

### 根因 B：四柱表头标题被 i18n 硬编码为"排盘 · 乾造"

无论 `chart.meta.genderTitle` 是什么，UI 都显示"乾造"。

**修复**：拆分 i18n key 为 `bazi.chartTitlePrefix` + `bazi.gender.{male/female}Title`，标题改为 computed 动态拼接。

```typescript
const chartTitle = computed(() => {
  const prefix = t('bazi.chartTitlePrefix')
  const gt = props.meta.genderTitle
  if (!gt) return t('bazi.chartTitle')
  const titleKey = gt === '乾造' ? 'bazi.gender.maleTitle' : 'bazi.gender.femaleTitle'
  return `${prefix} · ${t(titleKey)}`
})
```

### 根因 C：BirthForm 中性别只在"开始排盘"时同步到 store

性别是 RadioGroup 的本地 `ref`，仅在 `onPaipan()` 一并 `userStore.update(...)`。这导致用户切换性别时既没 store 变化，也没触发任何重排，UI 表面"看不出区别"。

**修复**：在 `BirthForm.vue` 中，仅给"性别"加 `watch`：一改即同步 store。其它字段（年/月/日/时）保留"先填再点排盘"的提交语义。

```typescript
watch(gender, (v) => {
  if (userStore.birth.gender !== v) {
    userStore.update({ gender: v })
  }
})
```

配合 `BaziPage`/`ZiweiPage` 中已有的：

```typescript
watch(
  () => userStore.birth.gender,
  () => { if (chart.value) onPaipan() },
)
```

形成闭环：切性别 → store → 自动重排 → UI 三处一致更新。

## 4. 实测印证（Playwright）

测试输入：1990 年 5 月 20 日 午时（农历），分别取男 / 女。

| 四柱 地支 | 午 / 午 / 申 / 午 | 午 / 午 / 申 / 午 | ❌ 一致 |

| 五行强弱 | 木 0 火 3 土 3 金 2 水 1 | 木 0 火 3 土 3 金 2 水 1 | ❌ 一致 |

## 5. UI 增强（已上线）

为避免用户再次产生"看上去一样"的疑问，在两个模块的"大运/大限"模块顶部加了带 hint 的徽章：

| 模块 | 徽章文案（男 / 女） | hint |
|---|---|---|
| 八字大运 | `乾造 · 顺行` / `坤造 · 逆行` | 此模块按性别展开（四柱五行十神不受性别影响） |
| 紫微大限 | `男命 · 大限按性别展开` / `女命 · 大限按性别展开` | 紫微大限按「阳男阴女顺行、阴男阳女逆行」分流（十二宫与四化不变） |

- 国风主题：金色描边胶囊
- 简约主题：浅灰底胶囊

## 7. 已知遗留 / 后续建议

1. **`bazi.fortune.subtitle = "起运 5 岁 · 一生十运 · 蛇形铺陈"`**
   该字符串是 i18n 静态文案，无论实际起运岁数是多少都显示 5。建议改成基于 `chart.fortune.startAge` 动态拼接。
2. **繁中 / 英文中 `ziwei.daxian.*` 整段未本地化**
   目前组件用 `te()` 兜底，繁/英用户看不到徽章。建议补齐文案。
3. **更广义的"性别相关说明"**
   未来可在「五行强弱」「十神结构」上加一行小灰字"此模块不受性别影响"，与大运/大限的徽章形成对照，进一步降低用户疑惑。

## 8. 总结

> "男女同生辰，命相同，运不同。" — 这是中国传统命理的基本前提，并非 bug。

本轮的修改没有改变任何算法，只解决了三个 UI/交互层的可见性问题：

1. 首屏让用户看到真实数据，而非 mock；
2. 标题反映真实命主性别；
3. 切性别立即重排，让"运"的差异肉眼可见，并辅以徽章+ hint 主动告知用户"此模块按性别展开"。

经过 Playwright 实测，男 / 女在四柱、五行、十神等"命"层面完全一致，在大运起运、行运方向、当前所处大运等"运"层面差异显著。修改链路完整可用。
