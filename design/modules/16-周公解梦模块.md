# 周公解梦模块

> **所属文档**：方案设计 · 子模块  
> **版本**：v1.0

## 1. 模块定位

- **方法**：周公解梦词典 — 搜索梦境关键词 → 返回传统寓意 + 现代化建议
- **可行性**：21/25 分 🟡 可做（见可行性分析 §3.2 方法 17）
- **体验定位**：搜索型工具，"梦见 X 是什么意思？"即查即得

## 2. 数据结构

```typescript
type DreamCategory =
  | '动物篇' | '植物篇' | '自然篇'
  | '建筑篇' | '物品篇' | '生活篇'
  | '活动篇' | '神鬼篇' | '其他篇'

interface DreamEntry {
  id: number
  category: DreamCategory
  keywords: string[]       // 搜索关键词（"蛇"、"梦见蛇"、"大蛇"）
  title: string            // "梦见蛇"
  content: string          // 寓意解释（古文 / 白话）
  suggestion?: string      // 现代化建议（LLM 生成 + 审校）
}
```

## 3. 算法

### 3.1 模糊搜索（fuse.js）

```typescript
import Fuse from 'fuse.js'
import { DREAM_ENTRIES } from '../data/jiemeng/zhougong.json'  // 懒加载

const fuse = new Fuse(DREAM_ENTRIES, {
  keys: [
    { name: 'keywords', weight: 2 },
    { name: 'title',    weight: 1.5 },
    { name: 'content',  weight: 0.5 },
  ],
  threshold: 0.3,   // 相似度阈值（0=完全匹配，1=全通过）
  includeScore: true,
})

export function searchDream(query: string, limit = 10): DreamEntry[] {
  if (!query.trim()) return []
  const results = fuse.search(query, { limit })
  return results.map(r => r.item)
}
```

### 3.2 分类浏览

```typescript
export function getDreamsByCategory(category: DreamCategory): DreamEntry[] {
  return DREAM_ENTRIES.filter(e => e.category === category)
}
```

## 6. 知识库来源

| 层 | 类型 | 来源 | 规模 |
|----|------|------|------|
| 词条数据 | B 类 | Hugging Face `hwl985/fortune-telling` 或 CSDN 公开数据集 | 2000-9000 条 |
| 分类体系 | B 类 | 传统《周公解梦》9 大类 | — |
| 现代化建议 | C 类（可选）| LLM 为高频词条（top 500）生成补充建议 + 审校 | 仅 top 500 |

**获取步骤**：
1. 从 Hugging Face 下载 `hwl985/fortune-telling` JSON → 转为 `DreamEntry[]`
2. 人工清洗（去重、修正编码）
3. 分类打标（用 9 大类）
4. LLM 为 top 500 高频词条生成现代化建议 + 审校
5. 最终压缩为 `zhougong.json`（~200 KB gzip 后 ~60 KB）

## 8. 风险

| 风险 | 应对 |
|-----|------|
| 搜索性能（9000 条 × 多字段模糊）| fuse.js 索引化；超过 3000 条时考虑 web worker |
| 数据集错漏（编码、重复）| 抽样校验 + 去重脚本 |
| 迷信引导风险 | 每条都显示："梦境解读仅为文化参考，不作为行为指导" |
| 敏感词（死亡、宗教）| 简单替换词典过滤；谨慎显示 |
