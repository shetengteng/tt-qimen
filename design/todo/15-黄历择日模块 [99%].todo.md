## 4. 跨模块依赖

| 依赖项 | 状态 | 说明 |
|--------|------|------|
| `useDivinationStore`（按月 hash 缓存） | `[~]` | 设计决议（2026-04-25）：通用 store 是单 slot key/data 槽（适合"一次输入 → 一组结果"场景），不适合月历"按 YYYY-MM 多 entry 命中"用法；在 `huangliStore` 内自建 Map LRU(6)，详见 §6 R1 |
| `useShareCard` / `ShareToast` | `[x]` | 已接入 |
| `useSkeletonReveal`（可选） | `[~]` | 设计决议（2026-04-25）：本模块同步计算 < 5ms，骨架反而会让首屏感知变慢，故**不接入**；如未来切换异步预计算可再启用 |

> **不依赖** BirthInput — 本模块按"日期"工作，与生辰无关。

## 6. 2026-04-25 · R1/R2/R3 三项尾活落地

### 6.1 R1 · 月历 LRU 缓存

| 项 | 决策 |
|----|------|
| 不接 `useDivinationStore` | 该 store 是「一组输入 → 一组结果」的单 slot 设计，月历"按月 key 多 entry"不匹配 |
| 缓存放在哪 | `src/modules/huangli/stores/huangliStore.ts` 内自建 Map |
| 容量策略 | LRU(6) — 当前月 ±2 月 + 用户最近翻过的 1 个月 |
| 命中时行为 | `delete + set` 重排顺序，确保最近访问者保留 |
| 失效时机 | 不主动失效（黄历静态数据，无 TTL 必要）；提供 `clearMonthCache()` 给测试 |
| 接入点 | `MiniCalendar.vue` 把 `getHuangliMonth()` → `store.getMonthCached()` |
| 暴露的诊断字段 | `_cacheStats()` 返回 `{ hits, misses, size, keys }`，仅测试用 |

**验证**：`scripts/verify-huangli-r1-cache.py` — 模拟 翻 7 个月 → cache.size == 6（容量上限），最旧 key 被淘汰；同一 key 二次访问 hit++ 且 keys 顺序后移；引用相等性（同一对象，未重新构造）。

### 6.2 R2 · 吉神凶神三语译表

| 项 | 决策 |
|----|------|
| 数据范围 | 遍历 1990-2030 共 ~2000 个采样日穷举 tyme4ts，去重得 60 吉神 + 91 凶神 |
| zh-TW 来源 | `chinese-conv` 程序化转换 + 专名人工修正（如「天后」不应作「天後」、「冲」校为「衝」） |
| en 格式 | `Pinyin (Translation)` —保留拼音便于懂术数的英文读者识别本名，括号内提供西方语境短译 |
| 兜底策略 | `translateGod()` 找不到时回原 zh-CN 名，UI 不会出现 undefined |
| locale 收敛 | vue-i18n 的 locale code 通过 `godLocale` computed 收敛到 `'zh-CN' / 'zh-TW' / 'en'` 三档 |
| 渲染入口 | `ShenshaCards.vue`（吉神/凶神条带）+ `DayDetailCard.vue`（详情 Dialog 内 tag 列表）|

**验证**：`scripts/verify-huangli-r2-i18n-gods.py` — 三语切换后 ShenshaCards 与 DayDetailDialog 内 tag 同步生效；`translateGod()` 抽样 5 个全译；繁体含「陽陰龍寶倉醫義」之一；英文含括号 + ASCII 字母。

### 6.3 R3 · 时辰吉凶圆盘可视化

| 项 | 决策 |
|----|------|
| 形态 | 12 等分圆盘（CSS `transform: rotate`），每段一个 button |
| 显示字符 | 取地支首字（如「子」）— 注意 `tyme4ts` 的 `h.name` 是「甲子时」三字结构，必须 `charAt(1)` 而非 `charAt(0)`，否则 12 个全显示天干会重复重叠 |
| 颜色编码 | 黄道 = `is-huangdao`（金色／紫色）；黑道 = `is-heidao`（红／灰）|
| 当前时辰 | `isToday=true` 时按浏览器本地时间高亮所在格 `is-current` |
| 中心区 | 默认显示"此时"或当日吉时摘要；点格后切换到所选时辰详情（名/干支/星/黄黑道）|
| a11y | `role="radiogroup"` + 12 个 `role="radio"` + `aria-label` + `aria-checked` + `aria-live="polite"` |
| 兼容降级 | DayDetailCard 内保留 `<details class="hl-dd-shichen-fallback">` — 折叠时不占空间，展开为旧文本列表 |

**验证**：`scripts/verify-huangli-r3-dial.py` — 12 cell 全在；当前时辰格有 `is-current`；点击任一格弹出 detail；ARIA 属性齐全；fallback `<details>` 存在且默认折叠。
