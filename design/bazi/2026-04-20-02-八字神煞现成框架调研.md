## 1. 总览与结论

**有，且有成熟的 TypeScript 插件。强烈推荐 `@lunisolar/plugin-char8ex`。**

### 1.1 候选框架对比

| 框架 | 四柱神煞 | 胎元/命宫 | 十神 | 纳音 | 空亡 | 每日神煞（黄历） | TS | 许可 | NPM 周下载 |
|---|---|---|---|---|---|---|---|---|---|
| **`tyme4ts`**（当前已用） | ❌ | ❌ | ✅ | ✅ | ❌ | ⚠️ 部分 | ✅ | MIT | 2.5K |
| **`lunar-typescript`**（6tail 同作者） | ❌ | ✅ | ✅ | ✅ | ✅（getDayXunKong） | ✅ | ✅ | MIT | 18K |
| **`lunisolar` + `plugin-char8ex`** ⭐ | ✅ | ✅ | ✅ | ✅ | ✅ | ✅（200+ 神煞） | ✅ | MIT | 主包 N/A，插件 79 |
| `mystilight-8char` | ✅（自称完整） | ? | ✅ | ✅ | ? | ❌ | ❌（纯 JS） | ? | 未知 |
| `chxb/shensha`（仅 GitHub，无 NPM） | ✅（51 种） | ❌ | ❌ | ❌ | ❌ | ❌ | ❌ | MIT | — |

### 1.2 关键洞察

1. **`tyme4ts` 本身不做四柱神煞**，它定位是"公历/农历/干支/节气"底层库，上层神煞要自建或引入别的库。
2. **`lunar-typescript`**（同作者的另一个库）也**不提供四柱神煞**，它的"神煞"是指"黄历每日吉凶神煞"（不同概念）。
3. **`@lunisolar/plugin-char8ex`** 是唯一一个 **NPM 上标准维护 + TS 类型完整 + API 干净**的四柱神煞插件，覆盖天乙贵人、文昌贵人、驿马、桃花、华盖、将星、劫煞、金舆、天厨贵人等常见神煞。

|---|---|
| NPM 包名 | `@lunisolar/plugin-char8ex`（依赖 `lunisolar` + `@lunisolar/plugin-takesound`） |

| 零生产依赖 | ✅（lunisolar 主包） |

### 2.2 API 速览

```typescript
import lunisolar from 'lunisolar'
import { char8ex } from '@lunisolar/plugin-char8ex'

lunisolar.extend(char8ex)

// 创建八字对象 (0=坤造/女，1=乾造/男)
const c8 = lunisolar('1990-05-20 12:26').char8ex(1)

// ===== 基础四柱 =====
console.log(c8.toString())         // 乾造: 庚午 辛巳 己未 庚午
console.log(c8.me.name)            // 日主天干（"己"）
console.log(c8.year.name)          // "庚午"

// ===== 十神 =====
console.log(c8.year.stemTenGod.name)         // 年柱天干十神
console.log(c8.month.branchTenGod.map(g => g.name))  // 月柱地支藏干十神数组

// ===== 纳音 =====
console.log(c8.year.takeSound)     // "路旁土"

// ===== 空亡 =====
console.log(c8.missing.map(b => b.name))  // ["戌", "亥"]

// ===== 四柱神煞 ⭐ =====
console.log(c8.year.gods.map(g => ({
  name: g.name,              // "天乙貴人"
  key: g.key,                // "TIAN_YI_GUI_REN"
  luck: g.luckLevel,         // 1=吉 / -1=凶 / 0=不定
})))
// [
//   { name: '天乙貴人', key: 'TIAN_YI_GUI_REN', luck: 1 },
//   { name: '文昌貴人', key: 'WEN_CHANG_GUI_REN', luck: 1 },
//   { name: '驛馬',    key: 'YI_MA',            luck: 0 },
//   { name: '桃花',    key: 'TAO_HUA',          luck: 0 },
// ]

// ===== 胎元 / 命宫 / 身宫 =====
console.log(c8.embryo().name)      // 胎元干支
console.log(c8.ownSign().name)     // 命宫干支
console.log(c8.bodySign().name)    // 身宫干支
```

**亮点：**
- API 极简（3 行引入 + 1 行取 gods 数组）
- 每个神煞自带 `luckLevel`（吉凶分档），不用自己判断
- 支持"男女神煞有差异"的场景（初始化时传性别）
- 类型定义完整（`.d.ts` 全覆盖）

### 3.3 核心函数

```typescript
// src/modules/bazi/core/shensha.ts
import lunisolar from 'lunisolar'
import { char8ex } from '@lunisolar/plugin-char8ex'
import zhCn from '@lunisolar/plugin-char8ex/locale/zh-cn'
import type { BirthInput } from '@/stores/user'
import type { ShenshaHit } from '../types'
import { SHENSHA_MEANING } from '../data/shenshaMeaning'

lunisolar.extend(char8ex)
lunisolar.locale(zhCn)

export function detectShensha(birth: BirthInput): ShenshaHit[] {
  const iso = toIsoString(birth)
  const c8 = lunisolar(iso).char8ex(birth.gender === 'male' ? 1 : 0)

  const list: ShenshaHit[] = []
  const pillars = [
    { name: 'year' as const,  pillar: c8.year },
    { name: 'month' as const, pillar: c8.month },
    { name: 'day' as const,   pillar: c8.day },
    { name: 'hour' as const,  pillar: c8.hour },
  ]

  for (const { name, pillar } of pillars) {
    for (const god of pillar.gods) {
      const meaning = SHENSHA_MEANING[god.key] || { short: '', long: '' }
      list.push({
        name: normalizeToSimplified(god.name),
        key: god.key,
        pillar: name,
        category: god.luckLevel === 1
          ? 'auspicious'
          : god.luckLevel === -1
            ? 'inauspicious'
            : 'neutral',
        short: meaning.short,
        long: meaning.long,
      })
    }
  }

  return dedupeAndSort(list)
}

function toIsoString(b: BirthInput): string {
  const mm = String(b.month).padStart(2, '0')
  const dd = String(b.day).padStart(2, '0')
  const hh = String(b.hour).padStart(2, '0')
  const mi = String(b.minute).padStart(2, '0')
  return `${b.year}-${mm}-${dd} ${hh}:${mi}`
}

function normalizeToSimplified(s: string): string {
  // 简单的繁→简映射，如"貴→贵"、"財→财"、"傷→伤"
  // 建议用 zhconvert 或自建 50 字映射表
  return s  // 占位
}

function dedupeAndSort(list: ShenshaHit[]): ShenshaHit[] {
  const seen = new Set<string>()
  return list
    .filter(h => {
      const k = `${h.pillar}-${h.key}`
      if (seen.has(k)) return false
      seen.add(k)
      return true
    })
    .sort((a, b) => {
      const ord = { auspicious: 0, neutral: 1, inauspicious: 2 }
      return ord[a.category] - ord[b.category]
    })
}
```

## 4. 风险与应对

| 风险 | 应对 |
|---|---|
| 引入 `lunisolar` 和 `tyme4ts` 两套底层库，有 ~30KB 增量 | 可接受：Vite tree-shaking 可裁到实际用到的部分；主页不会加载八字模块的代码（路由懒加载） |
| `lunisolar` 和 `tyme4ts` 对"节气换月"判断的算法细节可能微异 | 神煞的计算**不依赖节气换月**（只看干支组合），实际无冲突 |
| `lunisolar` 默认繁体中文 | 挂 zh-cn 语言包 + 加一层简体映射（50 字以内的常用繁→简） |
| `@lunisolar/plugin-char8ex` 更新频率低（2023 年 12 月最后一版） | ✅ 实际上八字神煞的规则是"固定不变"的古籍规则，库稳定是优点不是缺点；真有 bug 可以 fork 自己维护 |
| 神煞覆盖数量（~40）vs 我们原规划（12） | ✅ 更多是好事，UI 层可以做"常用 / 全部" tab 筛选 |
| 性能 | 神煞计算是 O(1) 查表，<1ms，不需要 Web Worker |

## 6. 决策建议

**推荐：采用 `@lunisolar/plugin-char8ex` 方案。**

优势：
1. ✅ 省去 40 个神煞的查表规则**推导 + 维护**（最容易出错的部分外包给成熟库）
2. ✅ 带吉凶分档 + 性别差异逻辑，符合古法
3. ✅ 未来可以用 `lunisolar` 的其他插件（如"吉方"、"胎神占方"、"黄历宜忌"）扩展**黄历模块**，一石二鸟
4. ✅ 纯前端静态，符合项目"无后端、无追踪"定位

劣势（可接受）：
1. ⚠️ 多一个底层库，Bundle 增 30KB
2. ⚠️ 需要做一层繁→简字符映射
