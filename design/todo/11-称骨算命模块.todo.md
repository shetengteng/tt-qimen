- 复杂度极低（4 个查表 + 1 次加法）；可行性 25/25 分 🟢 强推荐。

## 3. 子功能详情

### 3.1 算法（设计文档 §3）

| 步骤 | 状态 | 备注 |
|------|------|------|
| 公/农历归一化 → `SolarDay` → `LunarDay` | `[x]` | `core/chenggu.ts::toSolarTime()`，与 bazi 口径一致 |
| 取年柱六十甲子名 → `YEAR_WEIGHT` 查表 | `[x]` | `eightChar.getYear().getName()` → `YEAR_WEIGHT[name]` |
| 取农历月份 → `MONTH_WEIGHT` 查表 | `[x]` | `lunarDay.getLunarMonth().getMonth()` |
| 取农历日 → `DAY_WEIGHT` 查表 | `[x]` | `lunarDay.getDay()` |
| 取时辰 index（`(hour+1)/2 % 12`）→ `HOUR_WEIGHT` 查表 | `[x]` | `hourToBranchIndex()`，子时特殊处理 |
| 浮点容差比较（`Math.abs(a-b) < 0.01`） | `[x]` | `findPoem()` 内 |
| 异常态外抛 `FortuneError` | `[~]` | 目前抛普通 `Error`，UI 兜底为 `computeError` 卡片；待通用层 `FortuneError` 正式落地后切换 |
