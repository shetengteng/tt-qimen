## 0. 模块定位

日本五格剖象法（1918 熊崎健翁创立）— 将姓名按笔画拆为天/人/地/外/总五格。
- 可行性 21/25 分 🟡 可做。
- 体验定位：**实用工具型** — 用户测自己名字 / 起名参考。
- 2026-04-23：**主体完工** — `src/modules/xingming/` 下 types/core/data/stores/components/XingmingPage 全量交付；81 数理、5 格算法、笔画修正表、复姓白名单、双主题、三语 i18n 均已上线。

## 附录 H · 2026-04-25 NameInput 移动端二次修复 + 出生年下拉化

> 用户反馈："学生姓名模块 在移动模式下有问题，同时年 需要下拉选择"
> 排查发现附录 G 修复仅处理了"姓/名 grid 列宽"，但**未处理 input 自身在窄 grid cell 中的溢出问题** —— Mobile 320px viewport 下整张 form card 仍然横向超出视口。

### H.1 根因分析

| 表象 | 实际原因 |
|------|---------|
| 320 / 360 / 390 viewport 下 form card 右边沿超出屏幕，`名字` / `出生年` input 被裁断 | HTML `<input>` 默认宽度 ≈ 150px（基于 `size` 属性），双主题 `birth-form.css` 都没声明 `width: 100%`，所以 input 不受 `.ds-input-group` grid cell 约束，反向撑爆 cell → 撑爆 grid → 撑爆整张卡片 |
| Grid item 在 cell 宽度小于 input 默认宽时拒绝收缩 | `.ds-input-group` 默认 `min-width: auto`（grid item 默认行为），等于 input 的 intrinsic 宽度，必须显式 `min-width: 0` 才能解除 |
| 出生年用 `<input type="text" inputmode="numeric">`，需要用户手动输入 4 位数字 + 校验 | 与同项目八字 BirthForm 的 `<Select>` 体验不一致；移动端键盘弹出占用一屏，体验差 |

### H.2 修复

| 问题 | 修复 | 涉及文件 |
|------|------|---------|
| Mobile 320px input 撑爆 grid cell | scoped `.ds-input-group { min-width: 0 }` + `.ds-input-group input { width: 100% }`（仅本模块作用域，避免动通用层 birth-form.css 引发跨模块回归） | `components/NameInput.vue` |
| 出生年改下拉选择 | 引入 shadcn `<Select>` + `<SelectTrigger w-full>` + `<SelectContent max-h-72>`；年份选项 `currentYear → 1900` 倒序（与 BirthForm yearOptions 同款）；首项用 sentinel `__unspecified__` 表示"未指定"（reka-ui SelectItem 不允许空字符串 value） | `components/NameInput.vue` |
| `birthYear` 提交逻辑兼容 sentinel | `commitAndEmit` 中 `raw === UNSPECIFIED_YEAR \|\| raw === '' ? null : Number(raw)`，原 store schema `birthYear: number \| null` 不变 | `components/NameInput.vue` |
| placeholder 文案改为"可选 · 选择年份" / "Optional · pick a year" / "可選 · 選擇年份" | 三语 `xingming.placeholder.birthYear` 重写 + 新增 `xingming.birthYearOption.unspecified`（"未指定" / "未指定" / "Not specified"） | `locales/{zh-CN,zh-TW,en}.ts` |
