# 性能优化与依赖清单

> **所属文档**：方案设计 · 子模块  
> **版本**：v1.0

## 1. 性能优化策略

| 策略 | 实现 | 预期收益 |
|------|------|---------|
| 路由懒加载 | `() => import('./views/BaziView.vue')` | 首屏不加载非首页代码 |
| 数据懒加载 | 灵签 JSON、笔画库、解梦词条在首次进入模块时动态 import | 节省 400+ KB |
| tyme4ts 按需引入 | 只 import 需要的类（`SolarTime`、`LunarDay`、`EightChar` 等）| tree-shake ~30 KB |
| 图片优化 | 国风纹样用 SVG、emoji 用系统字体 | 节省图片体积 |
| 字体优化 | 中文字体走系统字体栈优先，CDN 为备选 | 不打包中文字体 |
| JSON 压缩 | 所有 data/*.json 使用 brotli 压缩 | 再节省 30% |
| 笔画库分表 | 常用 3000 字默认加载，生僻字按需 | 姓名学首次用 < 50 KB |
| 大运模板延迟加载 | `fortuneHints.ts` 仅在八字页面加载 | 首屏节省 20 KB |
| Service Worker（可选）| 缓存静态资源 + 模块数据 | 二次访问近零流量 |

## 3. 依赖清单

### 3.1 必需依赖

| 包名 | 用途 | 未压缩体积 | 许可 |
|------|------|-----------|------|
| `vue` | 框架 | ~35 KB | MIT |
| `vue-router` | 路由 | ~10 KB | MIT |
| `pinia` | 状态管理 | ~8 KB | MIT |
| **`tyme4ts`** | **命理核心**（农历/干支/八字/大运/流年/黄历）| ~80 KB | MIT |
| `chinese-character-strokes` | 康熙笔画（20000 字）| ~150 KB | MIT |
| `@vueuse/core` | 工具 hooks | ~15 KB | MIT |
| `fuse.js` | 模糊搜索（解梦）| ~6 KB | Apache 2.0 |
| `html2canvas` | 分享卡片导出 | ~43 KB | MIT |

**必需依赖体积总和**：~347 KB uncompressed / ~100 KB gzipped（不含业务代码和数据）

### 3.2 `tyme4ts` 核心 API 速查（MVP 全部使用）

| 需求 | API | 模块 |
|------|-----|------|
| 公历 → 农历 | `SolarDay.fromYmd(y,m,d).getLunarDay()` | 全部 |
| 公历 → 八字 | `SolarTime.fromYmdHms(...).getLunarHour().getEightChar()` | 八字 |
| **大运** | `EightChar.getYun(gender)` → `Yun.getDecadeFortunes(10)` | 八字 |
| **流年（任意年干支）** | `LunarYear.fromYear(2026).getSixtyCycle()` | 八字 |
| 节气 | `SolarYear.fromYear(2026).getJieQiList()` | 择日（可选）|
| 黄历宜忌 | `LunarDay.getRecommends() / .getAvoids()` | 择日 |
| 吉神凶神 | `LunarDay.getJiShen() / .getXiongShen()` | 择日 |
| 十二建星 | `LunarDay.getDayOfficer()` | 择日 |
| 时辰八字 | `LunarHour.getEightChar()` | 八字（子时换日依 tyme4ts 默认）|

## 4. 缓存策略

| 资源类型 | 缓存策略 | 备注 |
|---------|---------|------|
| HTML | no-cache 或短时缓存 | 确保 deploy 后立即更新 |
| JS/CSS（hash 文件名）| max-age=31536000（1 年）| Vite 默认带 hash |
| JSON 数据文件 | max-age=604800（1 周）| 内容稳定，更新频次低 |
| SVG 图标 | max-age=2592000（30 天）| 几乎不变 |
| Service Worker（可选）| 自管理 | 支持离线使用 |

## 5. 性能基线（目标值）

基于 Chrome DevTools Lighthouse Desktop + 4G 模拟：

| 指标 | 目标 |
|------|------|
| FCP (First Contentful Paint) | < 1.5s |
| LCP (Largest Contentful Paint) | < 2.5s |
| TTI (Time to Interactive) | < 3.0s |
| CLS (Cumulative Layout Shift) | < 0.1 |
| Lighthouse Performance | ≥ 90 |
| Lighthouse Accessibility | ≥ 95 |

## 6. 构建配置提示

### 6.1 Vite 配置要点

```typescript
// vite.config.ts
export default defineConfig({
  build: {
    rollupOptions: {
      output: {
        manualChunks: {
          // 核心库独立 chunk，跨页复用
          'vendor-core': ['vue', 'vue-router', 'pinia'],
          'vendor-tyme4ts': ['tyme4ts'],
        },
      },
    },
  },
  plugins: [
    vue(),
    visualizer({ open: false, filename: 'stats.html' }),  // 体积分析
  ],
})
```

### 6.2 部署压缩

- 主机提供 gzip / brotli 响应头
- 对 JSON、JS、CSS、SVG 开启压缩
- 预压缩（vite-plugin-compression）生成 `.gz` 和 `.br` 文件
