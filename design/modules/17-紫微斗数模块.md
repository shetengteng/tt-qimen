## 1. 模块定位

紫微斗数是 `tt-qimen` 的**核心专业支柱**之二（与八字并列为"命理双擎"）。

- **产品角色**：对标专业紫微站（如小藏等），让用户看到完整的 12 宫命盘 + 四化 + 大限；所有解读基于"模板化组合"的规则生成，避免 LLM 自由发挥的幻觉。
- **定位差异**：
  - **八字**：擅长"先天格局 + 十年一运 + 五行旺衰"，线性时间感强
  - **紫微**：擅长"12 宫人生板块 + 四化变数 + 主星坐落"，空间感强（一张盘看一生）
- **MVP 范围**：12 宫完整盘 + 14 主星 + 辅星 + 四化 + 命主身主 + 五行局 + 大限 + 流年
- **后续扩展**：飞星三合派、小限、流月流日、三方四正组合论、宫位两两对冲

## 2. 模块边界

| 维度 | MVP 做 | MVP 不做（后续扩展）|
|------|-------|-------------------|
| 12 宫盘 | ✅ 4×4 布局桌面盘 / 移动分宫展开 | 三合宫高亮联动 |
| 14 主星 | ✅ 名称 + 亮度（庙/旺/得/利/平/不/陷）| 主星两两组合论 |
| 六吉六煞 | ✅ 魁钺、昌曲、左辅右弼 / 擎羊陀罗火星铃星地空地劫 | 煞星入宫避煞口诀 |
| 杂耀 | ✅ 禄存、天马、红鸾、天喜等重要杂耀 | 所有次级杂耀 |
| 四化 | ✅ 本命四化（化禄/化权/化科/化忌）| 飞宫四化 |
| 大限 | ✅ 10 年一限 + 命宫切换 | 大限四化、小限 |

## 3. 数据结构

### 3.1 ZiweiChart（主要输出）

```typescript
interface ZiweiChart {
  solarDate: string            // 阳历 YYYY-MM-DD
  lunarDate: string            // 农历字符串
  chineseDate: string          // 四柱（年月日时干支）
  timeRange: string            // "03:00~05:00"
  sign: string                 // 星座
  zodiac: string               // 生肖
  gender: '男' | '女'
  
  // 核心元信息
  earthlyBranchOfSoulPalace: string  // 命宫地支
  earthlyBranchOfBodyPalace: string  // 身宫地支
  soul: string                       // 命主（北斗七星之一）
  body: string                       // 身主（南斗六星之一）
  fiveElementsClass: string          // 五行局（水二/木三/金四/土五/火六）
  
  // 12 宫位（按顺序：财帛/子女/夫妻/兄弟/命宫/父母/福德/田宅/官禄/仆役/迁移/疾厄）
  palaces: Palace[]
  
  // 本命四化
  mutagen: {
    hualu: string    // 化禄星
    huaquan: string  // 化权星
    huake: string    // 化科星
    huaji: string    // 化忌星
  }
  
  // 大限 / 流年 / 流月运限（懒加载）
  horoscope?: Horoscope
}

interface Palace {
  name: PalaceName           // 命宫/兄弟/夫妻/...
  heavenlyStem: string       // 宫位天干
  earthlyBranch: string      // 宫位地支
  isBodyPalace: boolean      // 是否身宫
  isOriginalPalace: boolean  // 是否来因宫
  
  majorStars: Star[]         // 主星 + 天马禄存
  minorStars: Star[]         // 六吉六煞
  adjectiveStars: Star[]     // 杂耀
  
  changsheng12: string       // 长生 12 神
  boshi12: string            // 博士 12 神
  
  stage: { range: [number, number], heavenlyStem: string }  // 大限
  ages: number[]             // 小限（MVP 仅存储，不展示）
}

interface Star {
  name: string
  type: 'major' | 'soft' | 'tough' | 'tianma' | 'lucun' | 'flower' | 'helper' | 'adjective'
  brightness?: '庙' | '旺' | '得' | '利' | '平' | '不' | '陷' | ''
  scope: 'origin' | 'decadal' | 'yearly'
}

type PalaceName =
  | '命宫' | '兄弟' | '夫妻' | '子女'
  | '财帛' | '疾厄' | '迁移' | '仆役'
  | '官禄' | '田宅' | '福德' | '父母'
```

### 3.2 ZiweiInterpretation（解读模板）

```typescript
interface ZiweiInterpretation {
  // 命宫主星论命（14 主星 × 2 性别 = 28 段）
  soulPalace: {
    star: string
    text: string        // 120-180 字
    keywords: string[]  // ['领导力', '孤高', '创业']
  }
  
  // 各宫主星简析（14 主星 × 12 宫 × 2 性别 ≈ 336 段，MVP 先做 14 × 12 = 168 段合并性别）
  palaceMajor: {
    [palaceName: string]: {
      [starName: string]: {
        text: string      // 60-80 字
        verdict: '吉' | '中' | '凶'
      }
    }
  }
  
  // 四化论命（10 天干 × 4 化 × 12 宫 = 480 段，MVP 精简为 10 × 4 = 40 段主轴解读）
  mutagen: {
    [stem: string]: {
      hualu: string
      huaquan: string
      huake: string
      huaji: string
    }
  }
  
  // 六吉六煞入宫（6 × 12 = 72 段 × 2 方向 = 144 段）
  minorStarImpact: {
    [palaceName: string]: {
      [starName: string]: string
    }
  }
  
  // 大限总评（60 甲子 × 吉凶标签 = 60 段）
  decadal: {
    [ganzhi: string]: { text: string, verdict: '吉' | '中' | '凶' }
  }
  
  // 当前流年（与大运共振，60 段）
  yearly: {
    [ganzhi: string]: { text: string, verdict: '吉' | '中' | '凶' }
  }
}
```

## 4. 算法与库依赖

### 4.1 核心库

- **`iztro`** v2.5.8（MIT、3.5K ⭐、原生紫微排盘）
  - `astro.bySolar(date, timeIndex, gender)` → 完整 12 宫 + 主星 + 辅星 + 四化
  - `astrolabe.horoscope(new Date())` → 大限 + 流年 + 流月 + 流日 + 流时
  - `star.getHoroscopeStar(stem, branch, scope)` → 独立获取流耀
- **无需自研**：所有排盘、亮度、四化推导均由 `iztro` 处理；我方仅做解读组合和 UI 呈现

### 4.2 流派设置

- **默认流派**：三合派 + 中州派（iztro 默认）
- **大限起法**：阳男阴女顺行，阴男阳女逆行（iztro 默认）
- **Post-MVP 可选**：飞星派（通过 iztro 的 `config` 插件切换）

### 4.3 性能策略

- **懒加载**：用户进入紫微页才按需加载 `iztro`（~60 KB gzipped）
- **Worker 排盘**：排盘操作置入 Web Worker（防止主线程卡顿）
- **缓存结果**：按 `solarDate + timeIndex + gender` 哈希缓存到 `localStorage`

## 5. 交互流程（与八字保持一致）

1. **生辰输入**（复用 `BirthInput.vue`）
2. **点击"开始排盘"** → 出现 1.5s 骨架屏
3. **结果区渐入**：
   - 元信息卡（命宫、身宫、五行局、命主、身主、四柱、生肖、星座）
   - **12 宫盘**（桌面 4×4 / 移动主宫 + 折叠其余）
   - **命宫主星论命**（120-180 字，带关键词芯片）
   - **四化总评**（四条，化禄/化权/化科/化忌 卡片）
   - **大限时间轴**（12 限，蛇形铺陈，与八字一致视觉）
   - **当前大限详情**（主星配合 + 四化共振）
   - **流年卡**（默认当年 + 可展开未来 10 年）
4. **分享 + 重新排盘**

## 9. 测试要点

- 阳历/农历输入一致性（调用 iztro `bySolar` / `byLunar` 结果对齐）
- 12 宫位顺序（命宫/兄弟/...）与 iztro 返回数据映射正确
- 亮度标签（庙/旺/得/利/平/不/陷）配色与图例一致
- 大限时间轴蛇形走向与八字一致
- 移动端主宫展示 + 点击展开其余 8 宫的交互顺畅
- 闰月修复逻辑（`fixLeap` 默认开启）

## 10. 风险与应对

| 风险 | 应对 |
|------|------|
| 12 宫盘移动端挤压 | 主要 4 宫（命/财/官/夫）默认展示 + 其余 8 宫折叠列表 |
| iztro API 变更 | 锁定 v2.5.x；升级前跑完整回归测试 |
| 解读模板量大（500 段）| 分批生成 + 人工审校 5-10 段/天；关键宫（命/财/官）先发版 |
| 同盘多流派差异 | 默认采用 iztro 默认流派；文档明确说明，Post-MVP 提供切换 |
| 紫微 + 八字双擎文案重复 | 共享"事业/财富/婚姻"词汇库，但解读角度拉开（八字从五行平衡、紫微从主星组合）|
| 首次加载体积大 | 懒加载 `iztro` + 解读 JSON 分文件 |
