# 八字模块 · 注释交互设计方案 (v3.1)

## 0 · TL;DR

### 本文档版本迭代

| 版本 | 形态 | 状态 |
|---|---|---|
| v1 | Popover + FAB + Sheet | 废弃（"点击后会中断"、移动端 Popover 被遮） |
| v2 | 全局 "展开全部注释" 开关 + 每项独立 Inline Expand | 废弃（按钮太重、inline 条目过于分散） |
| v2.1 | 章节级"注释"小按钮 + 单张合集卡 | 废弃（见 §10 反思） |
| v2.1.1 | 注释+折叠并列置右、等高 / 合集卡嵌入 section 内随折叠一起收起 | 废弃（见 §10 反思） |
| v3 | 就地内联展开（inline-annot）· 锚点触发 · 自由叠开 · 去卡片化 | 替换为 v3.1（就地会推挤子项，无全开按钮） |
| **v3.1** | **section 尾部聚合展开 + 展开全部 toggle + 标题式展开项** | **当前主推（详见 §11）** |

### 用户锁定的 v2.1 交互范式（引述原话）

1. "这个方案可以" — 指 v2 的"不打断、就地展开"方向保留；
2. "注释 button 简单一点，放在标题后面的 收起和展开 button 前面即可" — **全局 toolbar 删除，改为每个 section 标题旁的小按钮**；
3. "每个标题的注释单独沾卡" — **每个 section 的所有注释聚合到一张卡片里，而不是逐条 inline**。

### 本方案结论

- **三处注释统一：** 纳音 / 格局 / 神煞仍是同一套交互语言
- **触发器：** 每个 section（"命盘"、"命盘简析"、"神煞"）的折叠头里，在"收起/展开"按钮的**左侧**追加一个 `◎ xx注释` 小按钮
- **承载体：** 每个 section 一张独立的**合集卡**（`annot-card`），放在 section 顶部，默认 `hidden`
- **锚点互操作：** 点击 Nayin 单元格 / 格局 tag / 神煞 chip，自动打开所属 section 合集卡并高亮对应行（scrollIntoView）
- **非打断：** 合集卡就地展开在 section 流内，不浮层、不遮挡，符合"阅读不中断"
- **技术栈：** Vue `<Transition>` + 原生 CSS，无新依赖；reka-ui / shadcn-vue 暂不引入

## 1 · 背景与痛点

### 1.1 · 现状（v2 尝试后的状态）

v2 方案落地后发现 3 个实用性问题：

| 问题 | 表现 |
|---|---|
| 每项独立 inline 条目过于分散 | 9 个神煞就有 9 个独立展开区，用户难以横向比较 |
| 顶部"展开全部注释"开关太重 | 与"折叠"按钮视觉抢焦点，新用户不知二者关系 |
| 全局开关状态与单项状态的交互复杂 | 合并/覆盖语义难讲清 |

### 1.2 · v2.1 的核心洞察

> **折叠头是用户已经接受的"章节级"控件。**  
> 注释也是"章节级"信息（纳音 4 条 / 格局 4 条 / 神煞 ~10 条），天然适合与折叠头同层呈现；合并到一张卡里方便整体对照阅读。

### 1.3 · 设计约束（ui-ux-pro-max · High-Severity）

- **触控目标 ≥ 44×44px** — Nayin button 扩到 32px 高 × 44 点击区；神煞 chip 扩到 44px 高
- **可见焦点环** — `:focus-visible` 2px outline
- **`prefers-reduced-motion`** — 动画时长降为 0
- **轻量视觉权重** — 章节头注释按钮使用 ghost 风格，不喧宾夺主

## 2 · 方案对比

| 方案 | 交互 | 是否打断 | 是否喧宾 | 得分 |
|---|---|---|---|---|
| v1 · Popover+FAB+Sheet | tap 锚点 → Popover；FAB 打开 Sheet | **是** | 是 | 70 |
| v2 · 每项 Inline + 顶部全局开关 | tap 锚点 → 就地展开；全局开关 | 否 | **是**（全局开关抢焦） | 90 |
| **v2.1 · 章节头按钮 + 单张合集卡** | tap 章节头 `◎xx注释` → 开合集；tap 锚点 → 打开合集 + 高亮对应行 | **否** | **否**（按钮小、贴在折叠按钮旁） | **105** ⭐ |

### 3.2 · 状态机

```
┌─────────────┐   tap 章节头按钮          ┌──────────────┐
│ card hidden ├────────────────────────▶│ card visible │
│ btn:aria=   │                         │ btn:aria=    │
│  pressed    │                         │  pressed     │
│  "false"    │    再 tap / Esc         │  "true"      │
└─────────────┘◀────────────────────────┴──────────────┘
        ▲                                       │
        │                                       │ tap 锚点（nayin/tag/chip）
        │  card 内 item.is-focused              │
        │  自动清除                             ▼
        │                              ┌────────────────────┐
        └──────────────────────────────┤ card visible       │
                                       │ + focused-item 高亮 │
                                       │ + scrollIntoView    │
                                       └────────────────────┘
```

### 3.3 · 视觉 Token 对照

| Token | 国风 | 简约 |
|---|---|---|
| 章节头 `◎` 按钮描边 | `rgba(176,141,59,0.3)` | `var(--mn-border)` |
| 章节头 `◎` 按钮 press 背景 | `rgba(194,58,42,0.08)` | `var(--mn-accent-soft)` |
| 章节头 `◎` 按钮 press 文字 | `--gf-cinnabar` | `--mn-accent` |
| 合集卡背景 | `rgba(253,245,221,0.7)` | `--mn-surface` |
| 合集卡左彩条 | `--gf-cinnabar` 3px | `--mn-accent` 3px |
| 合集卡条目背景 | `rgba(255,251,240,0.8)` | `--mn-surface-soft` |
| focus-item 边框 | `--gf-cinnabar` + 2px glow | `--mn-accent` + 2px glow |

### 3.4 · 动效

| 动作 | 时长 | 曲线 | reduce-motion 降级 |
|---|---|---|---|
| 合集卡开合 opacity + translateY | 220ms | `cubic-bezier(0.22, 0.9, 0.24, 1)` | 0ms |
| focus-item 高亮过渡 | 180ms | ease | 0ms |
| scrollIntoView | 平滑 | browser 默认 | `block:nearest` 无动画 |

### 3.5 · 无障碍 checklist

- [x] 章节头按钮用 `<button type="button">` + `aria-pressed` + `aria-controls="<cardId>"`
- [x] 合集卡 `role="region"` + `aria-label="四柱纳音 · 合集"`
- [x] 锚点（Nayin/tag/chip）仍是 `<button>`；点击同时控制打开合集 + 高亮对应行
- [x] 键盘：Enter/Space 触发章节头按钮或锚点；Esc 关闭当前焦点所在的合集卡
- [x] focus-visible 2px outline；颜色契合主题
- [x] 合集卡 scrollIntoView 使用 `block: 'nearest'` 避免大幅跳动

### 5.2 · 新增 2 个通用组件

#### `src/components/ui/AnnotSectionToggle.vue`
```vue
<script setup lang="ts">
interface Props {
  pressed: boolean
  controls: string
  label: string
}
const props = defineProps<Props>()
const emit = defineEmits<{ 'update:pressed': [v: boolean] }>()
</script>

<template>
  <button
    type="button"
    class="annot-section-toggle"
    :aria-pressed="props.pressed"
    :aria-controls="props.controls"
    @click="emit('update:pressed', !props.pressed)"
  >
    <span aria-hidden="true">◎</span> {{ props.label }}
  </button>
</template>
```

#### `src/components/ui/AnnotCard.vue`
```vue
<script setup lang="ts">
interface AnnotItem {
  focus: string
  label: string
  short?: string
  long?: string
}
interface Props {
  id: string
  open: boolean
  title: string
  subtitle?: string
  items: AnnotItem[]
  footer?: string
  focused?: string  // 当前高亮的 focus key
}
const props = defineProps<Props>()
</script>

<template>
  <Transition name="annot-card">
    <div
      v-show="props.open"
      :id="props.id"
      class="annot-card"
      role="region"
      :aria-label="props.title"
    >
      <div class="annot-card-title">
        <span>◎ {{ props.title }}</span>
        <span v-if="props.subtitle" class="annot-card-title-sub">{{ props.subtitle }}</span>
      </div>
      <div class="annot-card-list">
        <div
          v-for="item in props.items"
          :key="item.focus"
          :data-annot-focus="item.focus"
          class="annot-card-item"
          :class="{ 'is-focused': item.focus === props.focused }"
        >
          <div class="annot-card-item-head">
            <span class="annot-card-item-key">{{ item.label }}</span>
            <span v-if="item.short" class="annot-card-item-short">· {{ item.short }}</span>
          </div>
          <p v-if="item.long" class="annot-card-item-long">{{ item.long }}</p>
        </div>
      </div>
      <div v-if="props.footer" class="annot-card-footer">
        <span>{{ props.footer }}</span>
        <span>共 {{ props.items.length }} 条</span>
      </div>
    </div>
  </Transition>
</template>
```

### 5.3 · BaziPage 调度

```ts
const annotState = reactive({
  nayin:   { open: false, focused: '' },
  pattern: { open: false, focused: '' },
  shensha: { open: false, focused: '' }
})

function toggleAnnot(kind: 'nayin'|'pattern'|'shensha') {
  annotState[kind].open = !annotState[kind].open
  if (!annotState[kind].open) annotState[kind].focused = ''
}

function focusAnnot(kind: 'nayin'|'pattern'|'shensha', focus: string) {
  annotState[kind].open = true
  annotState[kind].focused = focus
}
```

## 9 · v2.1.1 修订（按用户反馈）

> 2026-04-21 用户在原型预览后提出 2 条关键修订，已同步落地到 `design/prototypes/guofeng/bazi.html` 与 `design/prototypes/minimal/bazi.html`。

### 9.1 · 按钮对齐：注释 + 折叠并列置右，等高

#### 反馈原话

> "首先 button 需要和 展开 button 放在一起，都在右侧，高度要一样"

#### 问题

`.collapsible-head` 采用 `justify-content: space-between`：左边 `<h3>` 标题、右边 `.collapse-toggle`。
v2.1 初版把"注释"按钮直接 `insertBefore(collapseBtn)` — 在 flex 约束下结果是**标题和注释按钮成为一组，折叠按钮单独挤到右边**，视觉上不成对，且两按钮尺寸不一致。

#### 修订

### 9.2 · 合集卡位置：section 内部展开，随折叠一起收起

#### 反馈原话

> "展开的内容需要在原有的模块中展开，而非单独的块；点击其中一个模块，把整个内容都展开是不行的"

#### 问题

v2.1 初版把 `annot-card` 作为 section 的**前置兄弟节点** `sectionEl.parentNode.insertBefore(card, sectionEl)`。带来 2 个副作用：

1. 视觉上像一个**独立块**漂浮在 section 之前，不是"模块内部的注释"
2. **折叠语义破坏**：当用户点"收起"折叠整个 section 时，`annot-card` 不在折叠范围内，仍然显示 — 违反"模块内的东西跟着模块一起收起"的直觉

#### 修订

- **DOM 层**：改为 `sectionEl.insertBefore(card, sectionEl.firstChild)` — 合集卡作为 section `collapsible-body` 的**第一个子元素**，和其余内容在同一个折叠容器里。
- **高度同步**（关键）：`collapsible-body` 用 `max-height` 做动画。合集卡展开后 `scrollHeight` 变大，需要主动 re-measure：

  ```js
  function syncSectionHeight(sectionEl) {
    if (!sectionEl.classList.contains('collapsible-body')) return;
    if (sectionEl.classList.contains('collapsed')) return; // 折叠态不动
    sectionEl.style.maxHeight = 'none';
    sectionEl.style.maxHeight = sectionEl.scrollHeight + 'px';
  }
  ```

  触发点：
  1. 点"注释"按钮开/关合集卡
  2. 点 `annot-trigger`（锚点）导致合集卡从 `hidden` 变为可见
  3. 按 Esc 关闭合集卡

- **CSS 清理**：`.annot-card` 去掉了 `max-width` 和 `margin: 0 auto`，改用 `margin: 0 0 var(--space)` — 既然嵌在 section 内部，就继承 section 本身的宽度约束，不再需要独立的水平居中。

> 2026-04-21 用户二次反馈，v2.1.1 的章节级合集卡范式在 6 项痛点上被整体否决；
> 经结构化需求收敛后锁定「就地展开 + 两级文字 + 自由叠开」三参数，替换整个交互模型。

### 10.1 · v2.1 范式被否决的 6 项痛点

用户在「注释交互——请具体指出哪里不好」的多选问卷中全选了以下 6 项：

| # | 痛点 | 本质归因 |
|---|---|---|
| A | 合集卡本身太重（一条要读一整卡） | 聚合形态与读者「只想看这一条」的意图不匹配 |
| B | 阅读路径远（点锚点后合集卡在 section 顶部） | 视线要跳回上方，打断阅读 |
| C | 点一条全部展开 | 聚合形态天然带来的 collateral reveal |
| D | 章节头按钮不好看（和折叠按钮抢焦点） | 多控件并排本身引入装饰负担 |
| E | 合集卡视觉/层级不好看（2 列栅格、左彩条、页脚） | 卡片装饰对阅读任务是噪声 |
| F | 锚点可供性弱（不知道能点） | 当所有真正的揭示都被"章节头那个按钮"承载时，锚点降级为装饰 |

### 10.2 · 用户锁定的三个参数（引述选择）

| 参数 | 值 | 说明 |
|---|---|---|
| 展开位置 | **D · 混合就地** | Nayin → 四柱表**下方整块**（表内会撑破 4 列对齐）；格局 tag、神煞 chip → **锚点正下方**（grid 容器用 `grid-column: 1 / -1`，flex 容器用 `flex-basis: 100%`） |
| 信息密度 | **2 · 两级文字** | 短释加粗一行 + 长释正文一段，去掉卡片背景、左彩条、页脚、边框——只留一条 2px 的竖线作为语义分隔 |

### 10.3 · v3 设计决策对照表

| 维度 | v2.1.1（废弃） | v3（当前） |
|---|---|---|
| 触发器 | 章节头 `◎xx注释` 按钮 + 锚点 | **仅锚点**（Nayin / 格局 tag 或 mn-badge / 神煞 chip）。章节头按钮删除 |

| DOM 策略 | `annot-card` 作为 `collapsible-body` 的 firstChild | `inline-annot` 作为 trigger 的 **nextSibling**，同时生效 `grid-column: 1 / -1` 与 `flex-basis: 100%`（父容器按自身布局择其一） |

| 无障碍 | `role="region"` + `aria-labelledby` + `aria-live` | `role="region"` + `aria-label` + trigger `aria-expanded` |

### 10.4 · 技术要点

**1. 锚点差异化插入**

Nayin 位于 4 列 CSS Grid 的单元格（`.bazi-cell`）内，`bazi-cell-nayin` 按钮自身只是单元格里的文本；直接把展开块插到按钮后会被困在单元格内撑破栅格。策略：**把 Nayin 的"展开锚"上提到父 `.bazi-cell`**，展开块作为 `.bazi-cell` 的兄弟节点插入；由于 `.bazi-cells` 是 4 列 grid，`inline-annot` 的 `grid-column: 1 / -1` 自动占满整行，不再破坏对齐。

tag / chip 直接作为 trigger 的 nextSibling：`.interpret-tags` 是 flex wrap，`.shensha-list` 是 CSS Grid `auto-fill minmax(200px, 1fr)`，分别靠 `flex-basis: 100%` 和 `grid-column: 1 / -1` 达成"整行换行独占"。

**2. 与既有 `collapsible-body` max-height 动画的兼容**

既有折叠动画基于 `max-height: scrollHeight`，展开块插入/移除后 `scrollHeight` 会变，若不 re-measure 会出现「展开块被裁切」或「section 尾部留白」。v3 复用 v2.1 的思路，在每次 toggle 后调用 `syncCollapsibleHeight(host)` 向上查最近的 `.collapsible-body` 祖先并同步 max-height。

**3. 兼容原有 `__initAnnotCards` 调用点**

主脚本第 2179 行仍以 `window.__initAnnotCards()` 触发注释模块初始化。v3 保留同名函数（内部改为只绑事件委托 + Esc），避免修改主脚本；未来改名 `__initInlineAnnots` 时主脚本只改一行。

### 11.5 · 关键 JS 逻辑

```js
// 三个 section 的元数据
var SECTIONS = [
  { sel: '.chart-section'/*国风*/ || '.bazi-table'/*简约*/, label: '纳音注释', barClass: 'is-nayin', barId: 'inline-annots-bar-nayin' },
  { sel: '.interpret-section'/*国风*/ || '.interpret'/*简约*/, label: '格局注释', barClass: 'is-tag', barId: 'inline-annots-bar-pattern' },
  { sel: '.shensha-section', label: '神煞注释', barClass: 'is-chip', barId: 'inline-annots-bar-shensha' }
];

// 点锚点：在 section.lastChild = bar 里追加/移除一条 item
function toggleOne(triggerBtn) {
  var meta = matchMeta(triggerBtn);                 // 根据 closest 判断归属
  var section = document.querySelector(meta.sel);
  var bar = getOrCreateBar(section, meta);          // 首次调用时 append 到 section
  var focus = triggerBtn.dataset.annotFocus;
  var existing = bar.querySelector('.inline-annot-item[data-annot-for="'+focus+'"]');
  if (existing) { existing.remove(); triggerBtn.setAttribute('aria-expanded','false'); }
  else { bar.appendChild(buildItem(triggerBtn)); triggerBtn.setAttribute('aria-expanded','true'); }
  if (bar.children.length === 0) bar.hidden = true;
  syncToggleAllLabel(section, meta);                // 文案随状态
  syncCollapsibleHeight(section);                   // max-height 同步
}

// 一键 toggle：有任意开 → 全关；否则 → 全开
function toggleAll(section, meta) {
  var bar = getOrCreateBar(section, meta);
  var anyOpen = !bar.hidden && bar.children.length > 0;
  if (anyOpen) {
    bar.innerHTML = ''; bar.hidden = true;
    section.querySelectorAll('.annot-trigger').forEach(t => t.setAttribute('aria-expanded','false'));
  } else {
    section.querySelectorAll('.annot-trigger').forEach(t => {
      if (!bar.querySelector('.inline-annot-item[data-annot-for="'+t.dataset.annotFocus+'"]'))
        bar.appendChild(buildItem(t));
      t.setAttribute('aria-expanded','true');
    });
    if (bar.children.length) bar.hidden = false;
  }
  syncToggleAllLabel(section, meta);
  syncCollapsibleHeight(section);
}
```

`injectToggleAllButtons()` 在 `.collapsible-head` 中找到 `.collapse-toggle`，把它和新的 `.annot-toggle-all` 一起放进 `.collapsible-head-actions` 容器（若尚未创建则创建），保证两按钮等高并列。

interface Item { focus: string; label: string; short?: string; long?: string }

  layout: 'nayin' | 'tag' | 'chip'

  label: string
}
defineProps<Props>()

  <Transition name="inline-annots-bar">

        v-for="it in items"
