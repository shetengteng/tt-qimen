# 观音灵签模块

> **所属文档**：方案设计 · 子模块  
> **版本**：v1.0

## 1. 模块定位

- **方法**：观音灵签 100 签抽签 — 仪式感最强的模块
- **可行性**：24/25 分 🟢 强推荐（见可行性分析 §3.1 方法 3）
- **体验定位**：有"摇签筒"动画的抽签体验；结果展示签诗原文 + 解曰 + 仙机 + 分类指引

## 2. 数据结构

```typescript
type LingqianLevel = '上上' | '上吉' | '中吉' | '中平' | '中凶' | '下下'

interface LingqianItem {
  id: number          // 1-100
  level: LingqianLevel
  title: string       // 典故名，如"钟离成道"
  poem: string[]      // 四句签诗
  jieyue: string      // "解曰"白话
  xianji: string      // "仙机"分类指引
  topics: {
    family: string
    marriage: string
    career: string
    wealth: string
    travel: string
    health: string
  }
}
```

## 3. 算法

**纯随机**（带约束）：

```typescript
export function drawLingqian(lingqianData: LingqianItem[]): LingqianItem {
  const randomIndex = Math.floor(Math.random() * lingqianData.length)
  return lingqianData[randomIndex]
}
```

**高级：防止"连续抽到同一签"**

```typescript
const lastDrawn = useStorage('qimen.lingqian.lastId', 0)

export function drawLingqianSmart(data: LingqianItem[]): LingqianItem {
  const filtered = data.filter(x => x.id !== lastDrawn.value)
  const result = filtered[Math.floor(Math.random() * filtered.length)]
  lastDrawn.value = result.id
  return result
}
```

## 6. 知识库来源

| 层 | 类型 | 来源 | 规模 |
|----|------|------|------|
| 100 签数据 | B 类 | 免费 API（如 `api.t1qq.com`）一次性导出 + 公开网站爬虫清洗 | ~100 KB |
| 签文全部内容 | B 类 | 公版文献（无版权风险）| — |

**获取步骤**：
1. 从开放 API 一次性拉取 100 签全数据 → 存为 `guanyin.raw.json`
2. 抽样 10% 校对（检查错字、标点、格式）
3. 规范化为 `LingqianItem` 结构
4. 发布为 `guanyin.json`（gzip 后 ~30 KB）

## 8. 风险

| 风险 | 应对 |
|-----|------|
| 不同版本签文差异（如"关帝签 vs 观音签"）| 文档声明采用"观音灵签百度百科通用版" |
| 爬虫数据错漏 | 10% 抽样 + LLM 辅助格式校验 |
| 用户"疯狂抽签" | 限流：localStorage 记录抽签次数，超过 N 次提示休息（不强制）|
| 合规：宗教内容 | 首页和页脚始终显示免责声明 |
